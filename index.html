Welcome to the Open Cataclysmic Variable Catalog! The goal of this catalog is to act as a centralized, list of all identified and candidate Cataclysmic Variables found from various ongoing sky surveys and data repositories. If you use this data, please reference the cited sources of that data. Further information regarding the catalog can be found on the about page or the Research Note

Search by column instructions

To order the catalog by a category, navigate to the top of the column you wish to order by, and select the arrows once or twice that are to the right of the title. To order by most recent entries, follow the above steps for the discovery date column.

Search by location instructions

The objects in the catalog may be sorted by location. To activate the location filter, scroll to the bottom of the table and select the box next to “Observable from,” submit the coordinate location or select the observatory, and click “on” to enter a date of observation.

Using Comments

Comments may be added by selecting the pencil icon on the far right of the table. This will take you to the github issues page where all previous comments for that object are viewable. Submit your comment as an issue with the title being the name of the star. Include reference information in your comment.

Tips for searching: Individual column searches (the text boxes at the bottom of the table) support basic logical operators. Criteria can be comma separated to match multiple criteria, e.g. DN, Candidate in the type field would return entries that match one or the other. Surrounding a selection in double quotes will enforce an exact match, e.g. “N” will match N and not DN. Adding an exclamation point to your query will negate the query, e.g. !ASASSN in the name column will return all events without ASASSN in its name. For numeric columns basic limits and ranges are supported; for instance, to search for events with Magnitude less than 14.2, enter <14.2 into the Mag. column filter field at the bottom of the table. Our search syntax currently supports <, >, <=, >=, ranges (e.g. 14.2 – 18.4), and multiple simultaneous criteria (e.g. OR) when individual rules are comma separated (e.g. <14.2, 15.0 – 16.5).
